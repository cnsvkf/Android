### 동기

- 값을 받고 다음 행동을 취함

### 비동기

- 기다리지 않고 먼저 행동 후 값이 오면 처리 

-> ___대기하는 동안 다른 일 가능___

-> 대기 상태를 만들어야함

---

### Coroutine

- Thread는 무겁기에 가벼운 Coroutine을 Thread 위에서 실행
하는 방식으로 Thread를 효율적으로 사용

- ___코드 상태를 가진 객체___

-> 중단하여 상태 저장 후 나중에 이어서 가능

-> ___형식___

```kotlin
// 중간에 멈출 수 있는 함수
suspend fun test() {
    val a = 10
    // 2초 딜레이
    delay(2000)
    val b = 20
}
```

- 흐름이 아니기에 Thread와 다르게 중간에 멈출 수 있음

-> ___형식___
```
기존 방식 (Thread 중심)
Thread 1 → 작업 1
Thread 2 → 작업 2
Thread 3 → 작업 3
→ Thread 계속 증가 (비효율)

Coroutine 방식
Thread 1
  ├ Coroutine 1
  ├ Coroutine 2
  ├ Coroutine 3
→ Thread 적게 사용 (효율적)
```

---

### Thread

- 프로세스(한 개의 작업)에서 여러 작업 실행하기 위해 프로세스 안에 Thread를 만들어서 작동

- OS가 관리하여 무거움 + 실행흐름이기에 멈출 수 없음

- ___메인 액티비티가 메인 스래드 위에서 실행___

- 비동기 방식이지만 점유를 하기에 위험

___형식___

```
메인 스레드:
    서버 요청 시작 (다른 스레드에 맡김)
    → 계속 UI 동작
    + (작업 스레드에 맞기기에 비동기방식)

작업 스레드:
    서버 응답 받음
    → 메인 스레드에 결과 전달
```

---

- 스레드는 점유하면 전체 실행이 멈추기에,

    상태만 저장하고 실행을 끊었다가 재개할 수 있는 코루틴을 씀

- 하나의 Thread 위 여러 Corutine가 번갈아가면서 사용 가능

-> _예시_

```
Thread 1:
    Coroutine A 실행
    ↓ (delay 만나면 suspend)
    Coroutine B 실행
    ↓ (또 suspend)
    Coroutine C 실행
    ↓
    다시 A 이어서 실행
```
