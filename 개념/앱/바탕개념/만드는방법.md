| 상황              | 많이 쓰는 방식                                               |
| --------------- | ------------------------------------------------------ |
| **신규 프로젝트**     | Compose + Hilt + MVVM + StateFlow                      |
| **기존 오래된 프로젝트** | XML + Fragment + ViewBinding/DataBinding + Dagger/Hilt |
| **점진적 전환 프로젝트** | 기존 XML 화면 일부 유지 + 새 화면은 Compose                        |
| **작은 개인 앱/토이앱** | Hilt 없이 수동 DI 또는 간단한 Repository 직접 생성                  |
| **큰 회사/복잡한 앱**  | Hilt, 멀티모듈, Repository, UseCase, StateFlow 구조 많이 사용    |
