    Context는 **앱/화면/컴포넌트가 Android 시스템 기능과 앱 리소스에 접근하기 위해 사용하는 “접근권한 + 환경정보 객체”**다.

    즉, Context 자체가 모든 데이터를 들고 있는 “창고”라기보다는,
    “이 코드가 지금 어떤 앱/화면/환경에서 실행 중인지 알려주고, 그 환경을 통해 Android 시스템 기능에 접근하게 해주는 입구”

    에 가깝다.

### Context는 특정 기능 하나가 아니라, Android 앱이 시스템과 연결되기 위해 필요한 실행 환경 객체

## 사용 이유

    어느 앱의 DataStore에 저장할 건데?
    어느 패키지의 내부 저장소를 쓸 건데?
    이 앱이 그 저장소에 접근할 권한이 있나?
    Activity 기준으로 실행 중인가?
    Application 기준으로 실행 중인가?

->

```kotlin
context.dataStore 
```
으로 접근

            내가 코드에서 보는 context는 그냥 객체 하나처럼 보이지만, Android 입장에서는 “이 요청이 어느 앱/어느 컴포넌트 기준인지”를 구분하는 기준
