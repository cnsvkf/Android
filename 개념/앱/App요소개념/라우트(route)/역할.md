## Route는 화면과 ViewModel을 연결하는 중간 계층

| 역할                              | Route에 넣어도 됨? |
| ------------------------------- | ------------- |
| ViewModel 가져오기                  | 가능            |
| `collectAsStateWithLifecycle()` | 가능            |
| `uiState`를 Screen에 전달           | 가능            |
| Screen 이벤트를 ViewModel 메소드에 연결   | 가능            |
| 화면 이동 이벤트 처리                    | 가능            |
| 실제 UI 배치                        | 비추천           |
| 서버 통신 코드 직접 작성                  | 금지            |
| 비즈니스 로직 작성                      | 금지            |
| 데이터 검증 로직 많이 작성                 | 보통 ViewModel  |


# 흐름
    ViewModel
        - 상태 관리
        - 입력값 변경 처리
        - 버튼 클릭 처리
        - 서버 통신 요청
        - 성공/실패 판단

    Route
        - ViewModel의 uiState 수집
        - Screen에 uiState 전달
        - Screen 이벤트를 ViewModel에 연결
        - navigation 이벤트 처리

    Screen
        - 실제 UI 그리기
        - TextField, Button, Column, Row 등 배치
