#### AlertDialog는 표준 제목·내용·버튼 구조에 적합하고, _복잡한 입력 UI나 자유로운 배치가 필요하면 Dialog를 사용하는 것이 권장_ 한다.

#### Popup은 부모 컴포넌트를 기준으로 위치가 계산되는 _별도의 오버레이이다._

   

## Dialog

 - 스스로 열리고 닫히지 않는다.

```kotlin
var showDialog by rememberSaveable {
    mutableStateOf(false)
}

if (showDialog) {
    DialogContent()
}
```

-> 구조 

    showDialog = false
            ↓
    팝업 Composable이 Composition에 없음
            ↓
    버튼 클릭
            ↓
    showDialog = true
            ↓
    재구성
            ↓
    팝업 Composable이 Composition에 추가됨
            ↓
    닫기 버튼 클릭
            ↓
    showDialog = false
            ↓
    재구성
            ↓
    팝업 Composable이 Composition에서 제거됨
