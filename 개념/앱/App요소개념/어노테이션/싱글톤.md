| 구분                  | Hilt `@Singleton` | Kotlin `object`      |
| ------------------- | ----------------- | -------------------- |
| 생성 주체               | Hilt가 생성          | Kotlin/JVM이 생성       |
| 접근 방식               | `@Inject`로 주입받음   | `ObjectName`으로 직접 접근 |
| 의존성 주입              | 가능                | 거의 부적합               |
| 테스트 교체              | 쉬움                | 어려움                  |
| Android Context 사용  | Hilt가 안전하게 주입 가능  | 직접 들고 있으면 위험할 수 있음   |
| 실무 Repository/Store | 적합                | 비추천                  |
| 단순 상수/Util          | 과할 수 있음           | 적합                   |


Hilt Singleton은 DI 컨테이너가 관리하고, object는 Kotlin 언어가 직접 관리

-> __누가 만들고, 누가 전달하고, 교체__ 할 수 있는지가 완전히 다르다.

1. Hilt @Singleton은 “Hilt DI 컨테이너 안에서 하나만 만들어서 관리하는 객체”

2. Kotlin object는 “Kotlin/JVM 언어 차원에서 전역으로 하나만 만들어지는 객체”
