## 기본 방식

- ViewModel이 애니메이션을 직접 실행하는 게 아니라, ____ViewModel은 상태만 바꾸고, 애니메이션은 Composable에서 처리____ 하는 구조가 실무적으로 좋다.

    - ViewModel: expanded = true 로 상태 변경

    - Composable: expanded 상태를 보고 크기/투명도/위치를 애니메이션 처리

- ___상태가 바뀌면 UI가 애니메이션으로 따라가게___ 한다.

        상태 변경 → Compose가 재구성 → 애니메이션 API가 중간 값 계산 → 화면이 부드럽게 바뀜
        

#### Compose에서는 보통 직접 startAnimation() 같은 걸 호출하지 않습니다.

```kotlin
val alpha by animateFloatAsState(
    targetValue = if (isVisible) 1f else 0f,
    label = "alpha"
)
```

: 뜻

    isVisible 값이 바뀜
    → targetValue가 바뀜
    → Compose가 alpha 값을 0f에서 1f까지 부드럽게 바꿔줌

    ViewModel
    ↓ uiState 변경
    Route
    ↓ uiState 구독
    Screen
    ↓ 상태에 따라 UI 표시
    Animation API
    ↓ 상태 변화에 맞춰 자연스럽게 애니메이션 처리
