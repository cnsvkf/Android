## 개념

Navigation ↔ ViewModel ↔ Screen을 연결해주는 중간 Composable

| 위치           | 역할                             |
| ------------ | ------------------------------ |
| `NavHost`    | 어느 주소에서 어떤 화면으로 갈지 결정          |
| `HomeRoute`  | ViewModel 연결, State 수집, 이벤트 연결 |
| `HomeScreen` | UI만 그림                         |

    NavHost
    ↓ navigation callback 전달

    HomeRoute
    ↓ ViewModel state 수집

    HomeScreen
    ↓ 사용자 이벤트 발생

    HomeRoute
    ↓ ViewModel 메소드 호출

    HomeViewModel

ViewModel 책임:
- 상태 관리
- 데이터 요청
- 비즈니스 로직 처리
- 사용자의 이벤트를 받아 상태 변경

Route 책임:
- ViewModel 가져오기
- ViewModel 상태를 collect
- Screen에 uiState 전달
- Screen에서 발생한 이벤트를 ViewModel 메소드에 연결
- 화면 이동 콜백을 Screen에 전달

Screen 책임:
- 화면 그리기
- 사용자가 누른 버튼/입력 이벤트를 callback으로 알리기

## 흐름

    NavHost
        ↓
    Route 호출
        ↓
    Route가 ViewModel 사용
        ↓
    Route가 StateFlow 수집
        ↓
    Screen에 uiState 전달

-> ___Route가 ViewModel을 가져다 쓰는 구조___

\+ ViewModel과의 관계

Route가 상태를 “받아서” State로 바꾸는 게 아니라, __ViewModel이 가지고 있는 상태를 Route가 화면용 State로 “구독”__ 하는 것

    Screen에서 입력 이벤트가 발생한다
    Route가 그 이벤트를 ViewModel 메소드에 연결해 둔다
    ViewModel이 값을 저장한다
    Route가 ViewModel의 StateFlow를 구독한다
    Route가 구독한 uiState를 Screen에 다시 전달한다

\+ Screen과의 관계

Route와 Screen은 ___부모-자식 Composable 관계___

    LoginRoute = 부모 Composable
    LoginScreen = 자식 Composable
