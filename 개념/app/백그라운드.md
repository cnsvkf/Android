# 개념

- WorkManager 

WorkManager는 지연 가능하지만 안정적으로 실행되어야 하는 비동기 작업을 예약하는 API다. 네트워크 연결 같은 조건이 만족되면 작업을 실행하도록 맡길 수 있다.(___중요___ )

    - 서버에 임시 저장 데이터 업로드
    - 하루에 한 번 데이터 동기화
    - 네트워크 연결되면 실패한 요청 다시 보내기
    - 앱이 꺼져도 나중에 실행되어야 하는 작업

-> ___파일 구조___
```kotlin
worker/
  SyncPostsWorker.kt
  UploadPendingMessageWorker.kt
```
---

- Foreground Service

사용자가 “지금 이 앱이 뭔가 계속 하고 있다”는 걸 알아야 할 때 쓴다.

Foreground Service는 __상태바 알림을 반드시 보여줘야__ 하고, 사용자가 앱을 직접 보고 있지 않아도 계속 실행될 수 있다.

__Android 14 이상을 타겟팅하면 Foreground Service type도 명시__ 해야 하고, __위치/카메라/마이크 같은 경우 런타임 권한 조건도 만족__ 해야 한다.

    - 음악 재생
    - 운동 기록
    - 실시간 위치 추적
    - 통화
    - 파일 업로드 진행

-> ___파일구조___

```kotlin
service/
  LocationTrackingService.kt
  MusicPlayerService.kt
```

---

- AlarmManager

정확한 시간에 뭔가 해야 하면 쓴다.

AlarmManager는 __앱이 실행 중이 아니어도 특정 시간 기반 작업을 트리거__ 할 수 있다.

Android 12 이상부터 정확한 알람 권한이 민감해졌고, ___Android 14에서는 대부분 새로 설치된 앱에서 정확한 알람 권한이 기본 허용되지 않는다.___

    - 오전 7시에 알림
    - 수업 시작 10분 전 알림
    - 약 먹을 시간 알림
---
- Firebase Cloud Messaging(FCM) 

서버가 Android 앱에게 푸시 메시지를 보내게 해주는 시스템

일반적인 종료/백그라운드 상태에서는 알림 가능


    앱 실행 중
    → WebSocket으로 실시간 메시지 수신

    앱 백그라운드 / 종료 상태
    → FCM 푸시 알림으로 새 메시지 알림

    앱 다시 열림
    → REST API로 누락된 메시지 동기화
    → 다시 WebSocket 연결
