## 로그인 전

    [LoginScreen]
    사용자가 아이디 / 비밀번호 입력
            ↓
    [LoginViewModel]
    로그인 버튼 클릭 이벤트 처리
            ↓
    [LoginUseCase]
    로그인이라는 하나의 기능 실행
            ↓
    [AuthRepository]
    로그인 요청 + 토큰 저장 책임
            ↓
    [AuthApiService]
    Retrofit으로 서버 API 호출
            ↓
    [Server]
    accessToken, refreshToken 반환
            ↓
    [TokenDataStore]
    토큰을 로컬에 저장
            ↓
    로그인 성공 상태를 ViewModel로 전달
            ↓
    화면 이동    

## 로그인 후 관리    
    로그인 성공
    → 서버가 accessToken, refreshToken 반환
    → DataStore에 토큰 저장
    → 이후 API 요청마다 OkHttp Interceptor가 accessToken을 Header에 추가
    → accessToken 만료 시 서버가 401 반환
    → Token Refresh 로직이 refreshToken으로 새 accessToken 발급
    → 새 토큰 저장 후 원래 요청 재시도
    → 권한이 필요한 기능은 AndroidManifest + 런타임 권한 처리


## 토큰 구조

        사용자 로그인
        ↓
        서버가 accessToken + refreshToken 발급
        ↓
        앱이 TokenDataStore에 토큰 저장
        ↓
        일반 API 요청
        ↓
        OkHttp Interceptor가 accessToken을 Authorization 헤더에 붙임
        ↓
        서버 응답
        ├─ 200 성공 → 그대로 사용
        └─ 401 인증 만료 → OkHttp Authenticator 실행
                        ↓
                refreshToken으로 accessToken 재발급
                        ↓
                새 accessToken 저장
                        ↓
                원래 요청을 새 토큰으로 다시 시도
