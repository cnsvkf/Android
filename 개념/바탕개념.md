# 개념

### Scope
- 스코프 = 코루틴의 생명주기(Lifecycle)를 관리하는 영역

-> ___종류___

- viewModelScope : 뷰모델이 살아있는 동안 유지

- LaunchedEffect  
    - Composable 진입 시 자동 실행      
    - Composable이 사라지면 자동 종료
```kotlin
    @Composable
    fun MyScreen() {
    LaunchedEffect(Unit) {
        println("Composable 살아있는 동안 실행")
        }
    }
```

- rememberCoroutineScope

    - UI에서 suspend 함수를 실행하기 위함 + UI 동작 실행을 위해 사용(___UI동작이 시간이 걸리기에 사용___)

    - launch{ } 로 수동 실행
    
    - 이벤트(클릭 등)에서 코루틴 실행할 때 사용
    
```kotlin
@Composable
fun MyScreen() {
    val scope = rememberCoroutineScope()

    Button(onClick = {
        scope.launch {
            println("버튼 클릭 시 실행")
        }
    }) {
        Text("클릭")
    }
}
```

### Flow

- 데이터흐름
---
- MutableStateFlow : 값이 바뀔 때마다 자동으로 UI나 다른 코드에 알려주는 상태 저장 객체

    - observer 패턴 형식

-> ___같이 쓰이는 것___


---
- asStateFlow() : 읽기 전용

-> __예시__
```kotlin
private val _state = MutableStateFlow(0)
val state = _state.asStateFlow()
```

---

- collectAsState() : MutableStateFlow은 데이터흐름이기에 composable이 모름 따라서 collectAsState()로 값 수짐

-> __예시__
```Kotlin
val name by viewModel.name.collectAsState()
```
