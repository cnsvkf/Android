### 서버 값 돌려받는 방법

---
---
### API
-> 서버에게 무엇을 호출할 지 작성

---
---
```
View (UI)
   ↓
ViewModel (요청 처리)
   ↓
Repository (중간 관리자) // 뷰모델은 로직관리인데 API 쓰면 역할 혼동 때문에  중간관리자 통해서 쓴다
   ↓
Service (API 호출 함수) :  endpoint를 정의하고 실행
   ↓	
Base URL + Endpoint(API의 일부분 : API 안에 있는 하나의 주소(기능)) → 서버 요청 	// Base URL : 서버 공통 시작 주소
 예시) https://api.example.com/) 변하지 않음
    	// Endpoint (API 경로) : 특정 기능을 수행하는 세부 경로

   ↓
Server(백엔드) (JSON 응답) // 플랫폼(언어) 상관없이 코드 볼 수 있게 json 형태
   ↓
Model (DTO로 변환) // 다시 코드 원래대로 변환
   ↓
ViewModel // 데이터 변경 반영 (호출은 View → ViewModel / 데이터 흐름은 ViewModel → View)
   ↓
View (UI 업데이트) // 뷰가 뷰모델의 ‘상태’를 읽고 있어서 다시 적용
```
