---
### composable 

- composable { } : 매개변수가 들어올 시 실행(매개변수에 따라 실행하는 거 다름)

->  ___예시___
```kotlin
composable(route = Route.Intro.route) {
           BookOn(
                onClick = { navController.navigate(Route.WelCome.route) }
            )
        }


```

- BookOn  클릭 시 Route.WelCome.route(="welcome") 으로 감

- route(문자열) : 문자열에 따라 어떤 화면 실행할 지 등록

---

```kotlin
sealed class Route(val route: String) {
    object Intro : Route(route = "intro")
    object WelCome : Route(route = "welcome")
    object  FirstTimeText : Route(route = "firstTimeText")
}
```

-> route를 문자열로만 쓸 시 표기실수 방지 + Route class 내 object를 민들어서 Route 인 걸 알기 쉽게 함

---
```kotlin
object Intro : Route(route = "intro")
```

-> Route 클래스 상속 받음
여기서 ___route =___ 은 composable 내 __route__ 가 아니라 class Route의 매개변수
